  <action>
Create src/lib/aiDecisionEngine.ts with the following functions:

**evaluateUpgradeOptions(ctx: StrategicContext):**
- Filter equipment to: equipped items with level < 10
- For each, calculate value score: `bonus / upgradeCost` (ROI)
- Filter out items where `upgradeCost > availableCredits` (availableCredits = player.credits * (1 - reserve)), where reserve = aiConfig.resourceAllocation.reserve (per D-04, default 0.2 = 20%)
- Sort by value score descending
- Return array of { equipment, value: score }

**evaluateTargetOptions(ctx: StrategicContext):**
- Filter targets to: unlocked targets where difficulty <= player.level + 2
- For each, calculate value score: `rewards.credits / difficulty` (risk-adjusted return)
- Weight by riskTolerance: if riskTolerance > 0.5, favor higher difficulty targets; if < 0.5, favor lower
- Sort by value score descending
- Return array of { target, value: score }

**makeStrategicDecision(ctx: StrategicContext): AIDecision | null:**
- Calculate available credits: `ctx.credits * (1 - ctx.aiConfig.resourceAllocation.reserve)` (per D-04)
- Use priorities to decide action type:
  - If priorities.upgrades > priorities.operations AND available credits > cheapest upgrade cost:
    - Return upgrade_equipment decision with best ROI equipment
  - If priorities.operations > priorities.upgrades AND player.energy >= target energy cost:
    - Return start_operation decision with best value target
  - If player.skillPoints > 0:
    - Return allocate_skill decision (prioritize hacking skill)
- Reasoning format (per D-03): `Analyzing... {specific reasoning} → {action}`
  - Example: `Analyzing... Basic Laptop upgrade gives +5 bonus at 100 credits (ROI: 0.05), 2 upgrades affordable → Purchasing Basic Laptop`
  - Example: `Analyzing... Local Coffee Shop WiFi yields 50 credits at difficulty 1 (risk-adjusted: 50.0), energy sufficient → Starting data_breach`
- Confidence: based on value score relative to alternatives (0.3-1.0 range)

Replace gameStore.ts makeAIDecision() to use the new engine:
- Import makeStrategicDecision from aiDecisionEngine
- Build StrategicContext from current state: credits, creditsPerSecond (getCreditRate()), level, energy, skillPoints, equipment, targets, aiConfig
- Return makeStrategicDecision(ctx)

Per D-02: This is the rules-based part of hybrid. LLM strategic override comes in Plan 03.
Per D-04: Reserve 20% is enforced — AI never spends below reserve threshold.
Per D-03: Reasoning is summary format with specific numbers.
