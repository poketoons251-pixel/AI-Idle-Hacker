    credits: number;
    creditsPerSecond: number;

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→Edge Function | Game state sent to server; LLM response must be validated before execution |
| Edge Function→LLM API | API key protected server-side; rate limiting needed |
| LLM response→game state | LLM output is untrusted — must validate decision type, targetId, confidence range |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Tampering | LLM response validation | mitigate | Validate AIDecision type against enum, verify targetId exists in game state, clamp confidence to 0.0-1.0 range before execution |
| T-04-08 | Information Disclosure | Game state in Edge Function request | accept | Game state contains no PII — only game data (credits, equipment, targets). Safe to send to LLM. |
| T-04-09 | Denial of Service | LLM API rate limiting | mitigate | Add client-side rate limiting: max 1 LLM call per 30 seconds. Cache LLM responses for similar game states. |
| T-04-10 | Spoofing | Edge Function endpoint | mitigate | Use Supabase's built-in function invocation (supabase.functions.invoke) which handles auth. Do not expose raw function URL. |
| T-04-11 | Repudiation | AI decision audit trail | mitigate | recordAIDecision logs all decisions including LLM-sourced ones. Add "source: 'llm' | 'rules'" field to AIDecision for attribution. |
</threat_model>
