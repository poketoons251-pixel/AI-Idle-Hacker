`handleCommand` callback:
```typescript
const handleCommand = useCallback((input: string) => {
  const term = terminalRef.current; // Need access to xterm instance
  if (!term) return;

  // Write the command echo
  term.writeln('');

  // Add to Zustand history
  useGameStore.getState().terminal?.addToHistory?.(input);

  // Execute via registry
  const ctx: CommandContext = {
    term,
    store: useGameStore.getState(),
  };
  commandRegistry.execute(input, ctx).then(() => {
    // After command completes, write new prompt
    term.write('\r\n\x1b[1;32m$\x1b[0m ');
  });
}, []);
```

**IMPORTANT:** The XtermTerminal component from Plan 01 manages the Terminal instance internally. To access it in TerminalContainer, you need to either:
(a) Add a `terminalRef` prop to XtermTerminal that gets set to the Terminal instance, OR
(b) Use a callback ref pattern.

Approach (a) is cleaner. Modify XtermTerminal.tsx to accept an optional `onTerminalReady?: (term: Terminal) => void` prop that calls back with the Terminal instance after `term.open()`. Then TerminalContainer can store this in a ref.

Update XtermTerminal.tsx:
- Add `onTerminalReady?: (term: Terminal) => void` to props interface
- After `term.open(containerRef.current)` and `fitAddon.fit()`, call `onTerminalReady?.(term)`
- This does NOT change XtermTerminal's behavior for Plan 01's use case — it's additive.
