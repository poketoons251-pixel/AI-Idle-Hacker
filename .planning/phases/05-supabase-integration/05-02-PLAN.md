  <action>
Create src/lib/cloudSyncService.ts with the following exports:

1. **saveToCloud(gameState: Partial<GameState>): Promise<{ success: boolean; error?: string }>**
   - Get current session via `getAuthSession()` from supabaseAuth
   - If session is null or isAnonymous(session) is true, return `{ success: false, error: 'Must link account to save to cloud' }`
   - Extract playerId via `getUserId(session)`
   - Serialize gameState to JSON. Validate size < 100KB (per D-02). If larger, return error
   - Upsert to `supabase.from('game_saves').upsert({ player_id: playerId, save_data: serializedState, device_info: { userAgent: navigator.userAgent }, save_timestamp: new Date().toISOString() }, { onConflict: 'player_id' })`
   - The UNIQUE index on player_id (from Plan 01 migration) ensures last-write-wins per D-02
   - Return `{ success: true }` on success, `{ success: false, error }` on failure

2. **loadFromCloud(): Promise<{ success: boolean; data?: GameState; error?: string }>**
   - Get current session via `getAuthSession()`
   - If no session or anonymous, return `{ success: false, error: 'Must link account to load from cloud' }`
   - Query `supabase.from('game_saves').select('save_data, save_timestamp').eq('player_id', playerId).single()`
   - If no row found (PGRST116 error), return `{ success: true, data: null }` (no cloud save yet)
   - Parse save_data JSON and return as GameState
   - Return `{ success: true, data: parsedState, timestamp: save_timestamp }`

3. **checkSyncConflict(localTimestamp: number): Promise<{ hasConflict: boolean; cloudTimestamp?: string; localTimestamp: number; resolution: 'local' | 'cloud' }>**
   - Get cloud save timestamp via loadFromCloud (just the timestamp, not full data)
   - Compare cloud timestamp with localTimestamp (convert both to Date for comparison)
   - If difference > 5 minutes (300,000ms), return `{ hasConflict: true, cloudTimestamp, localTimestamp, resolution: 'cloud' }` (cloud wins per D-02 last-write-wins)
   - If difference <= 5 minutes, return `{ hasConflict: false, localTimestamp }`
   - If no cloud save, return `{ hasConflict: false, localTimestamp }`

4. **getCloudSaveTimestamp(): Promise<string | null>**
   - Lightweight query: `supabase.from('game_saves').select('save_timestamp').eq('player_id', playerId).single()`
   - Returns timestamp string or null

Import supabase from src/lib/supabase.ts. Import auth functions from src/lib/supabaseAuth.ts.

Use the GameState interface from src/store/gameStore.ts for type safety.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → game_saves table | Client sends full game state JSON; must be validated for size and structure |
| Anonymous → Cloud save | Anonymous users cannot save to cloud — prevents orphaned data |
| Conflict resolution | Last-write-wins means one device's state is lost; user must be warned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | saveToCloud | mitigate | Validate JSON size < 100KB; RLS ensures player can only upsert their own row via `auth.uid() = player_id` |
| T-05-08 | Information Disclosure | loadFromCloud | mitigate | RLS policy: SELECT only where `auth.uid() = player_id` — players cannot read other players' saves |
| T-05-09 | Denial of Service | idbStorage cloud hook | mitigate | Cloud save wrapped in try/catch — failures silently logged, local save always succeeds |
| T-05-10 | Repudiation | Conflict resolution | accept | Last-write-wins is standard idle game pattern; conflict warning informs user of potential data loss |
</threat_model>
