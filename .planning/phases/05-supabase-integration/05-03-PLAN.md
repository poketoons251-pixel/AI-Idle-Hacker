Create src/lib/leaderboardService.ts with the following exports:

1. **getLeaderboard(category: string = 'overall', limit: number = 50): Promise<LeaderboardEntry[]>**
   - Query `supabase.from('global_leaderboards').select('*').eq('category', category).order('score', { ascending: false }).limit(limit)`
   - Map results to LeaderboardEntry interface with rank computed from row position
   - Return array of entries

2. **submitScore(entry: { username: string; score: number; level: number; reputation: number; totalCredits: number; operationsCompleted: number }): Promise<{ success: boolean; error?: string }>**
   - Get session via getAuthSession(). Reject if anonymous
   - Get playerId via getUserId(session)
   - Upsert to `supabase.from('global_leaderboards').upsert({ player_id: playerId, username: entry.username, score: entry.score, level: entry.level, reputation: entry.reputation, total_credits: entry.totalCredits, operations_completed: entry.operationsCompleted, category: 'overall', last_updated: new Date().toISOString() }, { onConflict: 'player_id,category' })`
   - Return success/error

3. **getPlayerRank(category: string = 'overall'): Promise<{ rank: number | null; entry: LeaderboardEntry | null }>**
   - Get session and playerId
   - Query player's entry: `supabase.from('global_leaderboards').select('*').eq('player_id', playerId).eq('category', category).single()`
   - If no entry, return { rank: null, entry: null }
   - Query rank: count how many entries have score > player's score, add 1
   - Return { rank, entry mapped to LeaderboardEntry }

4. **subscribeToLeaderboard(category: string = 'overall', callback: (payload: RealtimePayload) => void): { unsubscribe: () => void }**
   - Create channel: `supabase.channel('leaderboard:' + category)`
   - Subscribe to postgres_changes: `.on('postgres_changes', { event: '*', schema: 'public', table: 'global_leaderboards', filter: 'category=eq.' + category }, callback)`
   - Subscribe: `.subscribe()`
   - Return { unsubscribe: () => channel.unsubscribe() }
   - Per D-03, this is the ONLY realtime subscription in v1

5. **updateLeaderboardEntry(updates: { score?: number; level?: number; reputation?: number; totalCredits?: number; operationsCompleted?: number }): Promise<{ success: boolean; error?: string }>**
   - Get session and playerId
   - Update: `supabase.from('global_leaderboards').update({ score: updates.score, level: updates.level, reputation: updates.reputation, total_credits: updates.totalCredits, operations_completed: updates.operationsCompleted, last_updated: new Date().toISOString() }).eq('player_id', playerId).eq('category', 'overall')`
   - RLS ensures player can only update their own entry

Import supabase from src/lib/supabase.ts. Import auth functions from src/lib/supabaseAuth.ts.

Define LeaderboardEntry interface locally (matching the one in Leaderboards.tsx page).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → global_leaderboards | Client submits scores; must validate score values are within reasonable bounds |
| Realtime → Client | Realtime pushes are public read; no sensitive data exposed |
| Leaderboard opt-out → Database | Player's privacy choice must be respected in leaderboard queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | Tampering | submitScore | mitigate | RLS: `auth.uid() = player_id` ensures players can only update their own entry; score values validated client-side (non-negative integers) |
| T-05-12 | Information Disclosure | getLeaderboard | accept | Leaderboard is intentionally public (per D-03); only shows username, level, score — no email, save data, or personal info |
| T-05-13 | Repudiation | Leaderboard submission | accept | Score submissions are idempotent (upsert); no audit trail needed for v1 |
| T-05-14 | Elevation of Privilege | updateLeaderboardEntry | mitigate | RLS policy: UPDATE only where `auth.uid() = player_id` — cannot modify other players' entries |
| T-05-15 | Denial of Service | Realtime subscription | accept | Supabase free tier supports concurrent realtime connections; monitor if usage grows |
</threat_model>
